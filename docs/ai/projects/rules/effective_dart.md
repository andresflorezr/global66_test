# Effective Dart & Linting

## Style Guide
- Follow the official **Effective Dart** guide.
- Adhere to `flutter_lints` rules (configured in `analysis_options.yaml`).

## Specifics
- **Types:** Use strict typing. Avoid `dynamic` unless absolutely necessary.
- **Naming:** Follow standard Dart naming conventions (camelCase for variables/functions, PascalCase for classes).
- **Comments:** Document public APIs and complex logic.
